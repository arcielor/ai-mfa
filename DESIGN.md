# Design System: SecureAuth

## 1. Overview

**Creative North Star: "The Clinical Vault"**

The SecureAuth design system balances a highly technical, dark-mode environmental backdrop with a stark, bright, and clinical focal surface. We prioritize the clarity of attack logs and functional flows over unnecessary visual flair. The interface feels bespoke and technically rigorous, avoiding generic off-the-shelf SaaS patterns. By placing clean white cards against a dark, animated grid, we draw immediate attention to the security actions and intelligence on display.

**Key Characteristics:**
- High contrast focal points against dark ambient space.
- Precision over decoration; every element serves a functional purpose.
- Intelligence on display; data and mono fonts are used to highlight ML components.

## 2. Colors

The palette is anchored by a stark contrast between deep environmental darks and crisp white surfaces, punctuated by a precise blue accent for interactive elements.

### Primary
- **Focus Blue** (#1d9bf0): The singular accent color. Used strictly for primary actions, focus rings, and critical links.

### Neutral
- **Deep Space** (#080d1a): The environmental background color. Sets the technical, terminal-like mood.
- **Clinical White** (#ffffff): The surface color for active panels and cards. Creates a focused, distraction-free zone.
- **High-Contrast Slate** (#0f1419): Primary text color on white surfaces.
- **Muted Slate** (#536471): Secondary text, labels, and helper text.
- **Crisp Border** (#cfd9de): Structural borders for inputs and containers.

### Named Rules
**The One Accent Rule.** Focus Blue (#1d9bf0) is the only decorative color on the login surface. It is reserved exclusively for interactive elements (buttons, links, focus states).

## 3. Typography

**Display Font:** Inter (with sans-serif)
**Body Font:** Inter (with sans-serif)
**Label/Mono Font:** JetBrains Mono (with monospace)

**Character:** Technical, legible, and modern. The pairing of a clean sans-serif with a strict monospace font reinforces the "intelligence on display" principle.

### Hierarchy
- **Display** (700, 24px, 1.2): Page titles and primary headers. Used sparingly to anchor the surface.
- **Body** (400, 15px, 1.5): Standard input text, button text, and paragraph content.
- **Label** (600, 11px, uppercase): Form labels and section headers. High contrast, small size, wide tracking.
- **Mono** (500, 12px, 1.5): Technical data, device fingerprints, and timestamps.

### Named Rules
**The Mono Intelligence Rule.** Any data generated by the ML engine or system (timestamps, IDs, scores) must be rendered in JetBrains Mono to distinguish it from user-provided content.

## 4. Elevation

The system uses a hybrid layering approach: a flat, dark environmental background with stark white, subtly shadowed central cards to focus attention.

### Shadow Vocabulary
- **Surface Shadow** (`box-shadow: 0 1px 2px 0 rgba(0, 0, 0, 0.05)`): A very subtle drop shadow used on the main authentication card to separate it slightly from the environment without looking "floaty".

### Named Rules
**The Grounded Surface Rule.** Cards should feel heavy and grounded. Avoid large, diffuse shadows that make elements look like they are floating far above the background.

## 5. Components

### Buttons
- **Shape:** Fully rounded pill (9999px radius).
- **Primary:** Focus Blue background with stark white text, padded generously (14px vertical).
- **Hover / Focus:** Background darkens slightly; transition is smooth and fast (200ms).
- **Secondary / Ghost / Tertiary (if applicable):** Text links in Focus Blue with underline on hover.

### Cards / Containers
- **Corner Style:** 16px radius (2xl).
- **Background:** Clinical White (#ffffff).
- **Shadow Strategy:** Surface Shadow (subtle 0.05 opacity).
- **Border:** 1px solid Crisp Border (#cfd9de).
- **Internal Padding:** Generous 32px all around.

### Inputs / Fields
- **Style:** 1px solid Crisp Border, Clinical White background, 4px radius.
- **Focus:** 1px solid Focus Blue with a 1px Focus Blue ring.
- **Error / Disabled:** High contrast red border/bg if error occurs.

### Navigation / Header Badges
- **Style:** Small pill shapes, white background, Crisp Border, Mono typography for technical data like live clocks.

## 6. Do's and Don'ts

### Do:
- **Do** use Focus Blue (#1d9bf0) strictly for interactive or primary focus elements.
- **Do** use JetBrains Mono for system-generated data and technical identifiers.
- **Do** wrap primary focal areas in a Clinical White card with a subtle Surface Shadow.

### Don't:
- **Don't** use generic Bootstrap or Material default templates.
- **Don't** use large, diffuse shadows on cards.
- **Don't** use gradient text or side-stripe borders as colored accents on cards.
- **Don't** use dark mode for the central authentication card itself (keep it Clinical White for precision).
